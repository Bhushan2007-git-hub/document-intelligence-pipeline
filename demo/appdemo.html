① When to choose Traditional ML
  • High-volume, low-latency pipelines
  • Well-structured, templated documents
  • Edge deployment (low memory)
  • Interpretability required
  • Classification only (no extraction)
② When to choose Transformer
  • Maximum accuracy is priority
  • Diverse, noisy, multilingual docs
  • Both classification + NER needed
  • Large annotated training set available
  • GPU infrastructure available
③ When to choose LLM-Assisted
  • Rapid prototyping / MVP phase
  • No labeled training data available
  • Document formats change frequently
  • Explainable decisions required
  • Low-resource deployment
